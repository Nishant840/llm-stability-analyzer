

# LLM Stability & Robustness Analyzer

A research-oriented framework for evaluating **Large Language Model (LLM) response stability under prompt variations**.

This project investigates how sensitive modern LLMs are to **changes in prompt phrasing**, and measures the **consistency, robustness, and contradiction behavior** of model outputs across different prompt styles.

The framework automatically generates prompt variations, collects responses from multiple LLMs, computes semantic similarity between responses, detects contradictions, and calculates a **stability score** to quantify model robustness.

An interactive dashboard is also provided to explore the results visually.

----------

# Motivation

Large Language Models often produce different answers when the same question is phrased differently. This phenomenon is known as **prompt sensitivity**.

For example:

```
Original: What is photosynthesis?
Expert: Explain photosynthesis as a biology expert.
Brief: Describe photosynthesis briefly.
Step-by-step: Explain photosynthesis step by step.

```

Even though these prompts ask the same question, the responses generated by LLMs can vary significantly.

This project aims to answer the following questions:

-   How stable are LLM responses when prompts change?
    
-   Do larger models produce more consistent answers?
    
-   How often do models contradict themselves across prompt variations?
    
-   Which prompt styles produce the most stable responses?
    

To explore these questions, we built a **complete evaluation pipeline for measuring LLM robustness**.

----------

# Features

This framework includes:

### Prompt Variation Testing

Automatically generates multiple prompt styles for the same question.

### Prompt Noise Robustness

Tests whether models remain stable when prompts include slight variations or noise.

### Multi-Model Evaluation

Responses are collected from multiple LLMs to compare stability across models.

### Semantic Similarity Analysis

Uses sentence embeddings to compute similarity between responses.

### Variance Analysis

Measures how much responses diverge across prompt variations.

### Worst-Case Similarity

Identifies the lowest similarity between responses to detect instability.

### Contradiction Detection

Detects whether responses contradict each other using Natural Language Inference.

### Stability Scoring

Computes a weighted stability score combining multiple evaluation metrics.

### Prompt Sensitivity Matrix

Analyzes which prompt styles produce the most consistent responses.

### Visualization

Generates charts and plots summarizing model robustness.

### Interactive Dashboard

A Streamlit dashboard allows interactive exploration of experiment results.

# Dataset Design

This project uses **10 manually curated questions**.

### Why only 10 questions?

The goal of this project is **not large-scale dataset evaluation**, but rather **methodology development for LLM robustness analysis**.

Each question generates multiple prompts and responses across several models.

Experiment size:

```
10 questions
6 prompt styles
5 models

```

Total responses collected:

```
10 × 6 × 5 = 300 responses

```

Pairwise similarity comparisons:

```
C(6,2) = 15 prompt pairs per question

```

Total similarity evaluations:

```
10 questions × 15 prompt pairs × 5 models
= 750 semantic similarity computations

```

Therefore, even with a small question set, the framework performs **hundreds of robustness comparisons**.

### Reason for using a smaller curated dataset

For a minor project, the objective is to demonstrate:

-   experimental methodology
    
-   evaluation framework design
    
-   model comparison techniques
    

Using a small but carefully designed dataset allows:

-   easier manual inspection of responses
    
-   clearer interpretation of stability results
    
-   faster experimentation during development
    

The framework is **fully scalable**, meaning it can easily be extended to larger datasets in future work.

# Stability Score

The final stability score combines several metrics:

-   Average semantic similarity
    
-   Worst-case similarity
    
-   Similarity variance
    
-   Contradiction rate
    

Example formulation:

```
Final Stability Score =
0.6 × average similarity
+ 0.2 × minimum similarity
− 0.15 × variance
− 0.05 × contradiction rate

```

Higher scores indicate **more consistent model behavior across prompt variations**.
